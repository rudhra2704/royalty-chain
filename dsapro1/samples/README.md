# RoyaltyChain Sample Files

This folder contains sample input and output files for project submission and evaluation.

## Folder Structure

```
samples/
├── input/                          # Reference data and test commands
│   ├── menu_commands.txt           # Menu choices to enter interactively
│   ├── license_records.txt         # All 16 license records
│   ├── account_keys.txt            # Hash table account data
│   ├── trade_volumes.txt           # Max-heap trade ranking data
│   └── routing_queries.txt         # Dijkstra routing test queries
└── output/                         # Captured program output
    ├── full_demo_output.txt        # Option 9 — all 8 data structures
    ├── license_search_output.txt   # Option 1 — B+ Tree search
    ├── revocation_log_output.txt   # Option 2 — Stack
    ├── transfer_queue_output.txt   # Option 3 — Queue
    ├── key_verify_output.txt       # Option 4 — Hash Table
    ├── trade_rankings_output.txt   # Option 5 — Max-Heap
    ├── dijkstra_output.txt         # Option 7 — Dijkstra routing
    └── huffman_output.txt          # Option 8 — Huffman compression
```

## How to Reproduce Output

```bash
make
./royaltychain
```

Then follow the commands in `input/menu_commands.txt`.

### Quick full demo

```
Enter choice: 9
Enter choice: 0
```

Compare terminal output with `output/full_demo_output.txt`.

### Individual module tests

| Test | Input file section | Output file |
|------|-------------------|-------------|
| License search | menu_commands.txt → Option 1 | license_search_output.txt |
| Revocation log | menu_commands.txt → Option 2 | revocation_log_output.txt |
| Transfer queue | menu_commands.txt → Option 3 | transfer_queue_output.txt |

## Note

Sample data is pre-loaded in `main.cpp` via `loadSampleData()`. The input files document what is stored in memory — they are not read from disk by the program. Output files were generated by running `./royaltychain` on the development machine.
